このファイルは `agents-OpenClaw/` の作業用タスクリストです。
公開フロントとつなぐ前提で、内部運用の安全性を保ちながら、案件化と処理可視化を支えるタスクだけを抜き出しています。

## Working Rule

- OpenClaw は内部専用の運用基盤として扱う
- 公開ユーザーから直接命令される前提で設計しない
- 公開に出す情報と内部だけで持つ情報を分ける
- GitHub Actions や Secrets は private 運用前提で、repo には入れない

### 追加したいフィールド案

```json
      "status": "受付済み",

```

## Priority 5

- [ ] 通知や同期のイベントから公開ステータスを更新できるようにする
- [ ] 手入力しなくても進行状況がある程度反映されるようにする
- [ ] ただし政治家・秘書の確認ポイントは残す

### 自動更新候補

- 案件作成時: `受付済み`
- 要点整理完了時: `確認中`
- 追加確認が必要な判定時: `追加情報待ち`
- スタッフレビュー開始時: `内部検討中`
- 返信送信時: `回答・案内済み`
- クローズ時: `完了`

### Done When

- 手動更新しなくても最低限の進行が見える
- 公開ユーザーが不安になりにくい
- 内部の判断を飛ばして自動確定しない

## Priority 6

- [ ] 公開面から直接叩かれない前提をコード上でも明確にする
- [ ] 外部入力をそのまま副作用処理へ流さない
- [ ] 将来 API 化しても安全境界を守れるようにする

### 守ること

- Gmail / Calendar / Tasks / Telegram を公開入力から直接操作しない
- 任意コマンド実行の口を作らない
- 公開 API と内部ジョブを分ける

### Done When

- 副作用のある処理が専用関数に閉じている
- 公開向け処理は read / draft 中心になっている

## Do Not Do

- Secrets を repo に書かない
- workflow 本番定義を public repo 前提で作らない
- 公開向け JSON に内部メモや個人情報を混ぜない
- Telegram 通知を公開ユーザー操作の直後に飛ばさない
