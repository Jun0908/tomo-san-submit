## プロンプト本文

```
以下のシステム概要をもとに、技術インフォグラフィック用のビジュアル構成案を作成してください。

---

## プロダクト名
Tomo-san — 政治家秘書 AI

## コンセプト
気仙沼市議会議員のともさんを支援する AI 秘書。
市民の声を受け取り、案件化し、議員に情報を届ける。

---

## システムの全体フロー（左から右、または上から下で図示）

1. 市民がブラウザで相談を入力
2. World Human Badge で人間確認（Bot 排除）
3. 会話 AI が相談内容を整理
4. OpenClaw（Claude API）が案件を自動生成
5. NEAR チェーンに案件レシート（ハッシュ）を記録
6. Telegram でともさんに朝ブリーフ・予定前ブリーフを送信
7. ともさんが Telegram コマンドで案件を検索・参照

---

## 技術スタック（カテゴリ別に色分けして表示）

### フロントエンド（青系）
- Next.js (App Router) + TypeScript
- 市民向け相談 UI
- Talking Head 動画プレイヤー

### AI・動画生成（紫系）
- Claude API（Anthropic）— 相談の案件化・返答生成
- SadTalker — 静止画から talking head 動画を生成
- edge-tts — テキストから音声合成（日本語）

### 信頼性・Web3（オレンジ〜緑系）
- World / Human Badge — 相談者が人間であることを確認
  - 使用場面：相談セッション開始時
  - 効果：Bot・重複相談を防ぐ
- NEAR Protocol — 案件レシートをオンチェーンに記録
  - 使用場面：OpenClaw が案件を生成した直後
  - 記録内容：案件ID・タイトル・要約ハッシュ・World 確認フラグ
  - 記録しないもの：生の相談テキスト・個人情報

### バックエンド（グレー系）
- Python（OpenClaw スクリプト群）
- Google Gmail / Calendar / Tasks API
- RSS（気仙沼市議会・市役所）
- Obsidian Markdown（ナレッジベース）

### 通知（黄色系）
- Telegram Bot API
  - 毎朝ブリーフ（予定・メール・案件サマリー）
  - 予定 30 分前ブリーフ（関連案件・確認事項）
  - /search /case /today コマンド

---

## 強調したい3つのポイント

1. **市民の声が「無視されない」**
   相談 → 案件化 → 公開ステータス表示 → ともさんへ通知

2. **信頼の三層構造**
   - World で「人間が送った」ことを保証
   - NEAR で「受理された」ことをオンチェーンに刻む
   - OpenClaw で「きちんと整理された」案件として残す

3. **ともさんの時間を守る**
   朝と予定前に必要な情報だけが届く。検索も Telegram から即座に。

---

## インフォグラフィックの要件

- A4 横向きまたは縦長スクロール型
- 日本語メイン、技術名は英語のまま
- フロー図（矢印あり）＋技術スタック表を組み合わせる
- World と NEAR は「信頼レイヤー」として目立たせる
- カラーパレット：落ち着いた和モダン系（深緑・紺・金）

上記の内容をもとに、インフォグラフィックのセクション構成・見出し文・各ブロックの説明文（1〜2行）を作成してください。
```
